# 📝 GoMemo 開発ロードマップ
 
温かみのあるモダンなメモアプリを完成させるためのタスクリストです。

---

## 🏗️ ステップ1：バックエンドの「口（API）」を作る
DBの準備は完了済み。Go側でデータを読み書きする窓口（エンドポイント）を実装します。

- [ ] **ユーザー認証の完成**
  - ログイン時にJWT（トークン）を発行し、誰がログイン中か識別できるようにする。
- [ ] **フォルダAPI**
  - フォルダの「作成・一覧取得・削除」機能を実装。
- [ ] **メモAPI**
  - メモの「投稿・一覧取得・編集・削除・ピン留め」機能を実装。

---

## 🔗 ステップ2：フロントエンドとバックエンドを「つなぐ」
React (Next.js) から実際のAPIを呼び出し、画面を動的にします。

- [ ] **共通フェッチ処理**
  - ログイン状態（トークン）をヘッダーに含めてAPIを叩く共通関数を作る。
- [ ] **フォルダの動的表示**
  - サイドバーにDBから取得したフォルダ名をリアルタイムに表示。
- [ ] **メモの動的表示**
  - ダミーデータを消し、DBから取得したメモをカード形式で表示。

---

## 🎨 ステップ3：操作画面（フォーム）の実装
ユーザーが実際にデータを入力・操作するUIを完成させます。

- [ ] **メモ作成モーダル**
  - 「メモを書く」ボタンで入力画面を出し、保存後に一覧を自動更新する。
- [ ] **フォルダ作成ダイアログ**
  - 新しいカテゴリ（フォルダ）を自由に追加できるようにする。
- [ ] **フィルタリング機能**
  - サイドバーのフォルダをクリックして、中身を絞り込めるようにする。

---

## ✨ ステップ4：仕上げ（クオリティアップ）
「最高にセンスあるUI」をさらに磨き上げ、使い心地を向上させます。

- [ ] **ローディング表示**
  - データ取得中に「ふわっ」と動くスケルトンアニメーションを導入。
- [ ] **レスポンシブ対応**
  - モバイル端末で見ても美しく、使いやすいレイアウトに調整。
- [ ] **バリデーション**
  - 空のメモや重複したフォルダ名などを防ぐエラーハンドリング。
